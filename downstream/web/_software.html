JBR Genome browser

Most existing genome browsers (e.g. UCSC genome browser, JBrowse, IGV, WashU) weren’t designed with extensibility in mind. As a result it is hard or impossible to customize or extend them. We’ve addressed the issue in the academic fashion by developing a browser of our own.

Some features highlights:

  • Supports semi-supervised peak calling using VPA?
  • Frontend-agnostic. The browser can be used on the desktop or as a web service.
  • Integrated support for common track formats, e.g. WIG, BED.
  • Integrated support for combining tracks, i.e. arithmetic operations.
  • Easy visualisation of multiple genomic loci
  • Custom statistics tracks for multiple regions available

VPA? Peak Caller

VPA? is a tool for analyzing and comparing ChIP-Seq data.
Both procedures rely on the Zero Inflated Negative Binomial Restricted Algorithm, see zinbra_scheme.pdf for details.

See releases for latest public release of peak caller. Nightly builds are available on teamcity CI.

Analysis

To analyze a single (possibly replicated) biological condition use analyze command. See details with command:

$ java -jar zinbra.jar analyze --help

The <output.bed> file will contain predicted and FDR-controlled peaks in the MACS2 format:

<chromosome> <peak start offset> <peak end offset> <peak_name> <score> . <coverage or fold/change> <-log p-value> <-log Q-value>

analyze reports p- and [Q-values] q for the null-hypothesis that a given bin is not enriched with ChIP-Seq modification. Peaks are formed from a list of truly (in the FDR sense) enriched bins for the analyzed biological condition by thresholding the Q-value with a threshold alpha and merging close peaks using gap option to broad ones. This is equivalent to controlling FDR at level alpha. If control is given it will be used with fragment_size to compute coverage for analysis.
